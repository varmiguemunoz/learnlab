## About this repository

This repository is my personal software engineering lab.

The goal is not to collect random exercises. The goal is to build small, clear, practical projects that help me understand how real systems work from the terminal to the backend, from scripting to APIs, from automation to scalable architecture.

Every folder in this repository represents a learning path. Some projects are simple experiments. Others are small utilities, command-line tools, backend services, frontend prototypes, or automation systems.

The common rule is that every project should teach something useful and be documented well enough for another developer to understand it, run it, and improve it.

This repository is designed for technical readers. If you are a developer exploring the code, you should be able to open any project and quickly understand what it does, why it exists, how to run it, and what concept it was built to practice.

## How each project is documented

Each project should include its own `README.md`.

The purpose of each project README is to explain:

```txt
What the project does
Why the project exists
What concept it practices
How to run it
What I learned while building it
```

A project should not only contain code. It should also contain reasoning.

## How to use this repository

Clone the repository:

```bash
git clone git@github.com:YOUR_USERNAME/learning-lab.git
```

Enter the repository:

```bash
cd learning-lab
```

Explore a language folder:

```bash
cd bash
```

Open any project and read its local `README.md` before running it.

Each project should include its own instructions.
